client에는 4개(datumo, ione, lablup, sktai)의 개별 feign client에 대해 상호간에 간섭여부를 확인하고 문제 있으면 수정해줘
- 각 feign client의 interceptor, config, errordecoder는 각 config의 file을 참조해야 함.
- interceptor, config, errordecoder에 @bean으로 등록되어 개별 서비스에 영향을 주면 안됨.

ione,elastic,lablup,udp feigin client도 sktai처럼 ErrorDecoder 개선해줘
- 응답 바디 파싱: 서버에서 반환하는 JSON 응답의 detail, message, error 필드를 파싱
- 422 오류 특별 처리: "Database error"가 포함된 경우 더 구체적인 메시지 제공
- 상세 로깅: 오류 상세 정보를 로그에 포함
